# คู่มือการ Deploy Node.js Application บน AWS ด้วย Terraform

ไฟล์ README นี้จะอธิบายขั้นตอนการนำโปรเจกต์แอปพลิเคชัน Node.js ไปรันบน AWS อย่างเป็นขั้นตอน ตั้งแต่การล็อกอินเข้า AWS CLI, การตั้งค่าคอนฟิกของ Terraform, ไปจนถึงการเข้าใช้งานเว็บไซต์

# ในไฟล์ main.tf บรรทัดที่ 97 - 101 เป้นข้อมูลละเอียดอ่อน ผมจะส่ง ข้อมูลให้ในทีม ส่วนตัวของอาจารย์ นะครับ

# สิ่งที่ต้องเตรียม (Prerequisites)
- บัญชี AWS ที่เปิดใช้งานแล้ว
- ติดตั้ง AWS CLI ไว้ในคอมพิวเตอร์
- ติดตั้ง Terraform ไว้ในคอมพิวเตอร์

# ขั้นตอนที่ 1: การตั้งค่า AWS Credentials ผ่าน AWS CLI

ก่อนที่ Terraform จะสามารถสร้าง Resource ต่างๆ บน AWS ได้ คุณจำเป็นต้องยืนยันตัวตนผ่าน AWS CLI เสียก่อน

1. เปิด Terminal หรือ Command Prompt
2. รันคำสั่งต่อไปนี้เพื่อตั้งค่า:
```bash
aws configure
```
3. ระบบจะให้ระบุข้อมูล 4 อย่าง ให้คุณกรอกข้อมูลของคุณลงไป:
- AWS Access Key ID: (ใส่ Access Key ของคุณ)
- AWS Secret Access Key: (ใส่ Secret Key ของคุณ)
- Default region name: ap-southeast-1 (สามารถใช้ Region นี้ตามที่กำหนดเป็นค่าเริ่มต้นในระบบได้เลย )
- Default output format: json

# ขั้นตอนที่ 2: การทำงานของไฟล์ Terraform
โปรเจกต์นี้ประกอบด้วยไฟล์คอนฟิก 3 ไฟล์หลัก ให้นำไปวางไว้ในโฟลเดอร์เดียวกัน:
- variables.tf: เป็นไฟล์ที่รวบรวมตัวแปรต่างๆ
- main.tf: เป็นไฟล์หลักที่ใช้ในการสร้างโครงสร้างทั้งหมดบน AWS
- outputs.tf: คอยจัดการการแสดงผล IP เครื่อง และ URL ของแอปพลิเคชันหลังจากที่สร้างระบบเสร็จสมบูรณ์


# ขั้นตอนที่ 3: เริ่มต้นใช้งาน Terraform และ Deploy
เมื่ออยู่ในโฟลเดอร์ที่มีไฟล์ทั้ง 3 แล้ว ให้เปิด Terminal แล้วรันคำสั่งตามลำดับดังนี้:
1. Initialize Terraform: เป็นการโหลด Plugins และเตรียมความพร้อมของ Terraform กับ AWS
```bash
terraform init
```
2. ตรวจสอบแผนการสร้าง: เพื่อดูว่า Terraform จะเพิ่ม หรือปรับปรุง Resource ใดบน AWS บ้าง
```bash
terraform plan
```
3. เริ่มการ Deploy: สั่งให้สร้างระบบขึ้นมาของจริง
```bash
terraform apply
```
ระบบจะแสดงรายละเอียดสิ่งที่จะถูกสร้าง ให้คุณพิมพ์คำว่า yes และกด Enter เพื่อยืนยันการทำรายการ

# ขั้นตอนที่ 4: การเข้าใช้งานแอปพลิเคชัน
หลังจากที่คำสั่ง terraform apply ทำงานจนเสร็จสมบูรณ์ ที่หน้าจอ Terminal จะแสดงค่า Output ขึ้นมา ให้สังเกตค่าเหล่านี้:
- เข้าชมเว็บไซต์: ให้คัดลอกค่าจาก app_public_url ไปเปิดใน Browser เพื่อดูหน้าเว็บ Node.js (ซึ่งจะอยู่ในรูปแบบ http://<Public-IP>:3000) 
- การเข้าไปดูแล Server (SSH): คุณสามารถดู Public IP ของ Server ได้จากค่า instance_public_ip หากต้องการเชื่อมต่อเข้าไปยังเครื่อง คุณสามารถใช้ไฟล์ Key Pair (เช่นไฟล์ .pem) ที่ Terraform เพิ่งสร้างมาให้ เพื่อเข้าไปยังระบบได้เลย 

# ขั้นตอนที่ 5: การลบระบบ (Clean Up)
เมื่อสิ้นสุดการใช้งานและไม่ต้องการเสียค่าใช้จ่ายให้ AWS เพิ่มเติม คุณสามารถลบ Resource ทุกอย่างที่สร้างขึ้นในโปรเจกต์นี้ได้ง่ายๆ ด้วยคำสั่ง:
```bash
terraform destryo
```
